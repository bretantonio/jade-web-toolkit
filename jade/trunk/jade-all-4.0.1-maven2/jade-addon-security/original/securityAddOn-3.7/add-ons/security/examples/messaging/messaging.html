Signing and encrypting messages

The following examples show how to make agents exchange signed and encrypted messages.
The two examples included in this section in particular show the sending part and the receiving part respectively and should be used together. In order to start them just launch a JADE main container with the security service, signature service and encrynption service and specifying the SingleUser authentication type as below

java jade.Boot -gui -services jade.core.security.SecurityService;jade.core.security.signature.SignatureService;jade.core.security.encryption.EncryptionService -jade_security_authentication_loginmodule SingleUser s:messaging.SecureSenderAgent r:messaging.SecureReceiverAgent

The JADE classes and the security add-on classes MUST be in the classpath.

SecureSenderAgent
Description:
This example is an agent that displays a simple GUI by means of which the user can create signed and/or encrypted ACLMessages. The interesting part of the example is the sendMessage() method of the SecureSenderAgent class where the jade.core.security.SecurityHelper is used to sign messages, retrieve the principal (basically the public key) of another agent and encrypt messages.
Jade classes used:
SecurityHelper
Source
/examples/messaging/SecureSenderAgent.java

 

SecureReceiverAgent
Description:
This example is an agent that waits for messages and printouts signed messages while discards the others. While printing a message it also says if the message was encrypted or not.
Jade classes used:
SecurityHelper
Source
/examples/messaging/SecureReceiverAgent.java
